SuitAndCape is an ever evolving project, allowing me to hone my web development and design skills by practicing familiar technologies, and trying out some new ones along the way. Here, you can find the projects I've made and get to know me a bit better along the way.
Currently, the site consists of four main sections.

Home: A standard landing page with the basic guide layout and a big ol' hello from yours truly.

About: Where I've been, what I'm up to, what I work with, why I do stuff. All wrapped up in one easy-to-find section. Neat

Projects: You can peruse some selections of things I have created solo or as part of a collaboration.

Flyer: An overview of the site's content. Also, the place I like to occasionaly stash Easter-eggs.

Notes

Throughout my "I work for the Internet" experience, I've had to learn a lot and I've had to learn it fast. The most effective way of accomplishing this, has been to build with the most basic form of the technology in question to get a deeper understanding of why it works the way it does. I whiteboarded the UX-workflow, wireframes, then decided on what tools to use.

This was the first time really working with Sass, so I decided to play with it on inferno-mode: use SMACCS methodologies with no frameworks or libraries (ie. Compass or Bourbon.io). Any helpful function or mixin snippets that I found online, I made sure to fully understand how they work.

I'm used to creating parallel back-end test suites, so because this project was really design heavy, I included a design-perspective test, which was highly beneficial to my mobile-first philosophy. Through the power of pure CSS, I created sliders, modals, and other features that usually rely on JavaScript.
